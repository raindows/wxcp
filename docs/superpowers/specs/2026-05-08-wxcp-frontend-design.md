# WXCP 企业微信营销平台 — UI/UX 设计文档

**日期**: 2026-05-08
**版本**: 1.0
**状态**: 已批准

## 1. 设计概述

面向专业运营团队的企业微信营销管理平台Web端，采用经典蓝色商务风格，注重操作效率与信息密度的平衡。基于 Vue 3 + Element Plus 技术栈实现。

## 5. 页面设计详情

### 5.1 工作台（首页）

路由: `/` 或 `/dashboard/workbench`

**布局从上到下:**
1. **4个核心指标卡片** (grid 4列): 今日新增客户、待跟进客户、进行中群发、今日消息发送量（含环比趋势箭头）
2. **两列布局**: 左侧待办事项列表（跟进提醒/待执行任务/过期SOP，最多10条）；右侧快捷操作入口（6个图标按钮）
3. **最近操作记录**（表格，最近5条）

### 5.2 客户管理

路由: `/customer/list`

**布局:**
1. 页面标题 + 操作按钮（同步客户、导出、添加标签）
2. 4个统计卡片: 客户总数、今日新增、待跟进、客户群数
3. 筛选栏: 关键词、跟进人、标签、添加时间
4. 表格列: 复选框、客户信息(头像+名称+企业)、来源、标签、跟进人、状态、添加时间、操作
5. 分页器（每页20条）

**客户详情** — Drawer 抽屉面板（640px）: 基本信息、标签管理、跟进时间线、互动记录

### 5.3 消息推送

路由: `/message/push`（发送页）、`/message/task`（任务列表）

**发送页:** 消息类型选择器 → 消息编辑区 → 发送范围选择器（左右分栏） → 底部操作栏

**任务列表:** 筛选栏 + 表格（任务名称、发送范围、类型、方式、状态、进度、时间、操作）

### 5.4 自动回复

路由: `/auto-reply/welcome`、`/auto-reply/keyword`、`/auto-reply/default`

Tab切换三个子页面，共享布局: 新增按钮 → 规则表格 → Dialog编辑

关键词回复特殊交互: 拖拽排序优先级、精确/模糊匹配切换

### 5.5 素材管理

路由: `/material/list`

**左右分栏布局:** 左侧分类树（240px）+ 右侧素材列表（卡片网格，4列）+ 拖拽上传区域

### 5.6 营销活动

路由: `/campaign/task`、`/campaign/sop`、`/campaign/journey`

**客户旅程** — 看板视图: 多列展示各阶段（新客→意向→成交），客户卡片支持拖拽流转

### 5.7 数据看板

路由: `/dashboard`

**布局:** 时间切换器 → 4指标卡片 → 客户趋势柱状图 + 员工排行 → 消息类型饼图 + 群发效果摘要

### 5.8 应用配置

路由: `/config/app`

应用列表（卡片形式）+ 新增/编辑 Dialog（corpId、corpSecret密码框、测试连接按钮）

### 5.9 通讯录管理

路由: `/contact/department`、`/contact/user`、`/contact/tag`

- 部门管理: 树形表格，支持新增子部门
- 用户管理: 左侧部门树 + 右侧用户列表
- 标签管理: 分组 + 标签列表

### 5.10 系统管理

路由: `/system/user`、`/system/role`、`/system/menu`、`/system/log`、`/system/config`

Tab切换五个子页面: 用户/角色/菜单/日志/配置，标准表格CRUD + Dialog编辑

## 6. 通用交互规范

### 6.1 操作模式

| 场景 | 交互方式 | 规格 |
|------|---------|------|
| 新增/编辑 | Dialog 对话框 | 宽度 520px~680px, 底部确定/取消 |
| 详情查看 | Drawer 抽屉 | 宽度 640px, 右侧滑出 |
| 删除确认 | MessageBox.confirm | 红色警告文字 |
| 批量操作 | 表格复选框 + 底部浮动栏 | 浮动栏 48px, 展示已选数量 |

### 6.2 反馈机制

| 类型 | 方式 | 持续时间 |
|------|------|---------|
| 操作成功 | ElMessage success | 2秒自动关闭 |
| 操作失败 | ElMessage error | 3秒自动关闭 |
| 加载中 | 按钮 loading + 骨架屏 | — |
| 空状态 | 插图 + 文字 + 操作按钮 | — |

### 6.3 表单规范

- 标签宽度: `80px` 右对齐
- 输入框高度: `32px`
- 必填项: 红色 `*` 标记
- 校验: 失焦时校验，提交时全量校验

### 6.4 表格规范

- 表头背景: `#f7f8fa`
- 行高: `48px`
- 悬停行: `#f7f8fa`
- 不使用斑马纹
- 操作列固定右侧

## 7. 响应式策略

| 断点 | 行为 |
|------|------|
| >= 1440px | 标准布局, 侧边栏 220px |
| 1200px ~ 1440px | 侧边栏可折叠为图标模式 (64px) |
| < 1200px | 不支持, 展示提示 "请使用桌面端访问" |

## 8. 图表组件规范

统一使用 ECharts，配色与设计系统一致:

| 图表类型 | 使用场景 | 配置要点 |
|---------|---------|---------|
| 折线图 | 客户趋势、消息趋势 | 主色填充面积, 平滑曲线 |
| 柱状图 | 月度对比、成员排行 | 主色 + 浅色双柱, 圆角顶部 |
| 饼图/环形图 | 消息类型分布 | 主色系渐变, 图例右侧 |
| 进度条 | 排行榜、任务进度 | 主色填充, 浅灰底色 |

图表容器高度统一 `240px`。
